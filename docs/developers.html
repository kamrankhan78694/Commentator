👨‍💻 Developer Setup

Complete guide to setting up your development environment for contributing to Commentator.

📋 Prerequisites

Node.js & NPM

Node.js 16+ and npm for package management and development tools.

node --version
npm --version

Ruby & Jekyll

Ruby 3.0+ and Bundler for Jekyll site generation.

ruby --version
gem install bundler

Git

Version control for managing code changes and collaboration.

git --version

🚀 Development Setup

1

Fork & Clone Repository

# Fork the repository on GitHub, then clone your fork
git clone https://github.com/YOUR-USERNAME/Commentator.git
cd Commentator
2

Install Dependencies

# Install Node.js dependencies
npm install

# Install Ruby gems for Jekyll
bundle install
3

Start Development Server

# Option 1: Jekyll development server
bundle exec jekyll serve --livereload

# Option 2: Simple npm development server
npm run dev
4

Create Feature Branch

# Create and switch to a new feature branch
git checkout -b feature/your-feature-name

# Make your changes and commit
git add .
git commit -m "Add your feature description"

🔄 Development Workflow

1. Making Changes

  • Edit files in your preferred code editor
  • Use the live reload server to see changes instantly
  • Test changes in multiple browsers
  • Follow existing code style and conventions

2. Testing Changes

  • Test in Chrome, Firefox, and Safari
  • Check mobile responsiveness
  • Verify Jekyll build works: bundle exec jekyll build
  • Validate HTML and CSS

3. Submitting Changes

  • Commit with descriptive messages
  • Push to your fork: git push origin feature/your-feature
  • Create pull request on GitHub
  • Respond to code review feedback

📝 Coding Standards

HTML/CSS

  • Use semantic HTML5 elements
  • Follow BEM-like CSS naming
  • Mobile-first responsive design
  • Accessibility best practices

JavaScript

  • Modern ES6+ syntax
  • Modular function design
  • Clear documentation
  • No external framework dependencies

Jekyll/Markdown

  • Use Jekyll front matter
  • Consistent layout structure
  • Clear documentation
  • SEO-friendly content
